# دليل هيكلية كود تطبيق مُبين (Life Planner)

بناءً على مبادئ **Ponytail Rules (الكسل الذكي)**، يهدف هذا الدليل إلى تفصيل بنية ملف `life-planner.html` (الذي يتجاوز 5400 سطر) لتسهيل فهمه وإجراء التعديلات عليه مستقبلاً بسرعة دون الحاجة لقراءة الكود بالكامل.

## 1. التقنيات والمكتبات المستخدمة (Dependencies)
- **الأساس:** HTML, CSS, Vanilla JavaScript (بدون أطر عمل معقدة مثل React أو Vue تطبيقاً لقاعدة استخدام الحد الأدنى).
- **قواعد البيانات والاستضافة:** Firebase (Auth, Realtime Database).
- **المكتبات الخارجية:** Chart.js (للرسوم البيانية)، html2pdf.js (لتصدير التقارير)، canvas-confetti (للاحتفالات البصرية).

---

## 2. بنية واجهة المستخدم (HTML & Layout Structure)
يعتمد التطبيق على مفهوم Single Page Application مبني بالكامل داخل ملف واحد. يتم التنقل عبر إخفاء وإظهار أجزاء من الكود.

### الصفحات الرئيسية (محددة بـ `class="page"`)
- `#page-daily` : لوحة التحكم الرئيسية (اليوميات، العادات المستحقة، لوحة المزاج).
- `#page-all-habits` : إدارة جميع العادات (خط 2017).
- `#page-goals` : إدارة الأهداف ومتابعة التقدم (خط 2044).
- `#page-reader` : المكتبة وتتبع قراءة الكتب (خط 2052).
- `#page-focus` : جلسات التركيز (نظام زها / بومودورو) (خط 2087).
- `#page-reports` : التقارير والإحصائيات الشاملة (خط 2536).

### النوافذ المنبثقة (Modals) (تبدأ من خط 2578)
- `#modal-settings`: إعدادات الحساب والمستخدم.
- `#modal-notif`: إعدادات التنبيهات.
- `#modal-habit`: نموذج إضافة/تعديل عادة.
- `#modal-goal`: نموذج إضافة/تعديل هدف.
- `#modal-book`: نموذج إضافة كتاب للقرأة.
- `#modal-book-notes`: تدوين ملاحظات حول كتاب معين.
- `#modal-zaha`: توثيق جلسة تركيز (زها).
- `#modal-breathe`: واجهة تمرين التنفس التفاعلي.

## 3. إدارة الحالة والبيانات (State Management)
يتم تخزين جميع بيانات المستخدم محلياً ومزامنتها مع Firebase في كائن واحد يسمى `state` (يبدأ تعريفه في السطر ~2859).

```javascript
let state = {
  habits: [],       // مصفوفة العادات
  goals: [],        // مصفوفة الأهداف
  books: [],        // مصفوفة الكتب المضافة
  zahaSessions: [], // توثيق جلسات زها (التركيز)
  journal: [],      // تدوينات اليوميات
  moods: [],        // متتبع المزاج
  nextId: 1,        // لتوليد معرفات فريدة للعناصر
  // ... متغيرات إضافية
}
```

### دوال المزامنة
- `loadState()`: جلب البيانات من Firebase وإعادة تحديث الواجهات.
- `saveState()`: حفظ كائن `state` بالكامل إلى Firebase.

## 4. خريطة الدوال البرمجية (JS Functions Map)
تبدأ أكواد الجافاسكريبت بعد السطر 2780. الدوال مقسمة وفق مسؤوليات واضحة:

### التنقل وعرض واجهة المستخدم
- `goPage(el)`: للتنقل بين الصفحات.
- `openModal(type, id)` / `closeModal(id)`: لفتح وإغلاق النوافذ المنبثقة.
- `toast(msg)`: لعرض الرسائل التنبيهية السريعة أسفل الشاشة.

### دوال العرض والتحديث (Render)
- `renderAll()`: دالة رئيسية تعيد رسم جميع عناصر التطبيق استناداً لبيانات `state`.
- `renderDailyMiniDash()`: تحديث واجهة قسم اليوميات (Dashboad).
- `renderHabits()` / `renderAllHabits()`: عرض قائمة العادات.
- `renderGoals()` / `renderBooks()`: تحديث قوائم الأهداف والكتب.

### دوال الإدارة والعمليات (CRUD Operations)
- **الحسابات (Auth):** `handleLogin()`, `handleSignUp()`, `handleLogout()`.
- **العادات:** `saveHabit()`, `delHabit()`, `toggleHabit(id)`.
- **الأهداف:** `saveGoal()`, `delGoal()`, `updateGoalUI()`.
- **الكتب:** `saveBook()`, `delBook()`, `updateBookPagesInline()`, `saveBookNotes()`.
- **زها (التركيز):** `saveZahaSession()`.

## 5. توافق الكود مع مبادئ (Ponytail Rules)
- **الحد الأدنى (Minimalism):** تم دمج كل التطبيق في ملف واحد للاستغناء عن إعدادات الـ Bundlers (مثل Webpack أو Vite) مما يجعل المشروع قابلاً للعمل مباشرة بفتحه في المتصفح.
- **عدم كتابة كود احتياطي (YAGNI):** لا توجد مكتبات قوية ومعقدة لمتغيرات بسيطة؛ استخدم التطبيق Vanilla JS لتعديل الـ DOM مباشرة.
- **استخدام ميزات النظام (Native):** التبديل بين الـ Dark/Light mode واستخدام المتصفح المحلي.

### كيفية التعديل في المستقبل بسرعة:
1. **لتعديل التصميم:** توجه للقسم العلوي من الملف (أسفل الـ `<style>`).
2. **لإضافة صفحة جديدة:** أنشئ `div` جديد بصنف `class="page"` في قسم الـ HTML، ثم أضف زراً للوصول له ينادي `goPage()`.
3. **لتغيير طريقة الحفظ:** عدل فقط دالتي `loadState()` و `saveState()`.
4. **لإضافة قسم جديد (مثل التمويل):** أضف مصفوفته الخاصة داخل كائن `state`، واكتب دالتي `renderFinance()` و `saveFinance()`.

## 6. سجل التحديثات الأخيرة (Recent Updates)
- **نقل المشروع:** تم فصل التطبيق بملفاته (`life-planner.html` ومجلد `sounds/`) في مجلد منفصل `life-planner` لضمان بيئة تطوير نظيفة.
- **إصلاح تقدم الأهداف (Goal Progress):** تم تعديل دالة `getGoalProgressInfo()` لتقوم بحساب تقدم الهدف بناءً على **كامل مدة الهدف** (حتى تاريخ نهايته) بدلاً من حساب الالتزام حتى اليوم الحالي فقط. هذا جعل نسبة التقدم الكلية تعكس الإنجاز الفعلي والمطلق نحو خط النهاية.
- **واجهة المستخدم:** إعادة إضافة وتجميل زر "إضافة مهمة `+`" في الصفحة الرئيسية بلون رمادي متناسق.
- **إدارة الأهداف:** تمت إضافة حقل لتحديد "تاريخ البداية" للهدف (افتراضياً يكون تاريخ اليوم)، بالإضافة إلى توجيه المستخدم تلقائياً إلى الصفحة الرئيسية بعد الانتهاء من حفظ الهدف.
