## 3. بيانات الأداء الحقيقية (مرجع للقرارات)

**LinkedIn:** الشخصي > التعليمي بـ 4x. ⇒ محتوى الألعاب التعليمي ينتمي لـ Instagram، لا LinkedIn.

**Instagram (ترتيب الأداء):**
1. كوميدي / موقف طريف — الأعلى وصولاً
2. قصة شخصية بتجربة حقيقية — تفاعل عالٍ
3. تقني بهوك فضولي ("كيف…؟") — يعمل جيداً
4. وراء الكواليس — يبني ارتباط
5. رأي جريء في المجال

## 4. الفورمولا الناجحة (مثبتة بالأرقام)

### بوست LinkedIn
```
Hook قوي (موقف مقلوب / جملة مفاجئة)
→ قصة شخصية بتفاصيل حقيقية
→ أرقام وتفاصيل ملموسة
→ درس أشمل
→ CTA بخيارات محددة (أ/ب/ج) — أقوى من "ما رأيك؟"
```
+ **أول تعليق جاهز يُنشر خلال 5 دقائق.**

### ريل Instagram (الهيكل المحسّن من التحليل)
```
هوك ٠-٢ ثانية (سؤال / موقف / حركة صادمة — لا "أهلاً بكم")
→ مشكلة (٢-١٥ث)
→ محتوى (١٥-٤٥ث)
→ خلاصة/CTA (٤٥-٦٠ث)
```
+ **9:16 كاملة** بلا أشرطة سوداء · عنوان نصي بارز على الثمبنيل · كابشن عامية أردنية/شامية · سؤال يفتح التعليقات · **٣-٥ هاشتاقات** (عربية + تقنية + ألعاب).

1. استبدل "١٢ سنة" بـ **"عقد كامل"** (تكرر كثيراً).
2. لا تضع بوستين عن الفشل/المصاعب في نفس الأسبوع.

6. **بصوت Blue Design:** عند الحديث عن الشركة استخدم لغة المنصة ء مكاني، الطبقات الثلاث) بدل "renders/استوديو تصميم" — لكن صوت صفوان الشخصي يبقى هو المحرّك للتفاعل.

## 9. طريقة العمل للجلسات الجاية (كيف تبني شهر جديد)

1. **اقرأ هذا الملف** + `صناعة المحتوى/CLAUDE.md` للسياق.
2. **أكّد المحور أولاً** مع المستخدم (المصدر/الموضوع/اللون) قبل البناء — شغل كبير.
3. لأي محتوى مرتبط بـ **Blue Design** شغّل سكِل `blue-design-marketing`. للأفكار الجديدة `pd-content-market-intelligence`. للبروفايل `pd-linkedin-profile-optimizer`.
4. **طريقة البناء المعتمدة (مهم):** مولّد بايثون:
   - `gen_monthN.py` = القالب (CSS/JS/HTML) — يُنسخ من مولّد شهر سابق عبر `sed` مع تبديل ألوان الثيم والعناوين.
   - `data_monthN.py` = البيانات: يضبط `g.WEEKS` و`g.WEEK_OF = [0]*8+[1]*7+[2]*8+[3]*7` و`g.DAYS` (٣٠ dict) ثم `g.build()`.
   - يُكتب المخرج في `محتوى للنشر/خطط_النشر_الشهرية/خطة_النشر_شهرN.html`.
5. **التحقق برمجياً (إلزامي):** ٣٠ day-card · ٦٠ badge-li · ٣٠ badge-ig · أيام 1-30 موجودة · توزيع 8/7/8/7 · دوال JS موجودة · لا `{{` متبقّي · RTL سليم.

### ⚠️ باگات معروفة في بيئة العمل
- **أداة Edit بتقطع الملفات الكبيرة** (~1700+ سطر) في النص وتفقد ما بعدها. ⇒ للملفات الكبيرة استخدم `head -N` للقص + `cat >> file <<'PYEOF'` (bash heredoc) لإعادة الإلحاق، وتحقّق بعد كل تعديل بـ `wc -l` + عدّ `"""` (لازم زوجي).
- **ملفات الكتاب العربية (.md في Book) ما بتنقرأ عبر bash** (تطبيع Unicode للأسماء). ⇒ اقرأها بأداة Read/Grep بمسار Windows (`F:\…`). ملفات HTML تُقرأ عبر bash عادي.
